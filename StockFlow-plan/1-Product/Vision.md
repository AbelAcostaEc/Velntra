# 📦 Velntra

> **Sistema de Inventario y Punto de Venta para pequeños negocios**

---

# Visión del Producto

## Descripción

Velntra es una aplicación web desarrollada para ayudar a pequeños negocios a administrar su inventario y registrar sus ventas de una manera sencilla, rápida y organizada.

El sistema busca reemplazar el control manual realizado en cuadernos o hojas de cálculo, permitiendo conocer en tiempo real el stock disponible, las ventas realizadas y el estado general del negocio.

La primera versión estará enfocada en un único negocio con una sola sucursal, priorizando la simplicidad, facilidad de uso y una buena experiencia para el usuario.

---

# Problema

Muchos pequeños negocios administran su inventario y ventas de forma manual.

Esto ocasiona problemas como:

- Desconocer el stock real de los productos.
- No saber cuánto se vendió durante el día.
- Errores al calcular el total de una venta.
- Dificultad para encontrar información de clientes.
- Pérdida de tiempo buscando productos.
- Falta de reportes para tomar decisiones.

Velntra nace para solucionar estos problemas mediante una plataforma web intuitiva y fácil de utilizar.

---

# Objetivo General

Desarrollar un sistema web que permita controlar el inventario y registrar las ventas de un pequeño negocio de manera segura, organizada y eficiente.

---

# Objetivos Específicos

- Registrar y administrar productos.
- Organizar productos por categorías.
- Registrar clientes.
- Registrar ventas.
- Actualizar automáticamente el stock después de cada venta.
- Consultar reportes básicos del negocio.
- Visualizar indicadores importantes desde un Dashboard.

---

# Público Objetivo

Velntra está dirigido a pequeños negocios como:

- Bazares
- Papelerías
- Librerías
- Tiendas de barrio
- Minimarkets
- Ferreterías pequeñas
- Tiendas de accesorios
- Negocios familiares

No está pensado para grandes empresas o cadenas comerciales.

# Filosofía del Proyecto

Velntra no busca convertirse en un ERP complejo desde su primera versión.

La prioridad es construir un producto pequeño, estable, intuitivo y bien desarrollado que resuelva correctamente las necesidades de un pequeño negocio.

Cada nueva funcionalidad deberá incorporarse únicamente cuando la versión actual esté completamente terminada y probada.

El crecimiento del sistema será incremental, manteniendo una arquitectura modular que facilite su evolución sin comprometer la calidad del código.
